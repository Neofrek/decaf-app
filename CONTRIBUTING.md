## 🚀 How to Contribute

To keep the development process clean and organized, please follow these steps:

### 1. Find or Open an Issue
Before writing any code, check the **Issues** tab to see if someone is already working on the bug or feature. If not, open a new Issue describing the changes you want to make so we can discuss the implementation.

### 2. Fork and Clone
1. Fork this repository to your personal GitHub account.
2. Clone your fork locally:
   ```bash
   git clone https://github.com
   ```

### 3. Create a Feature Branch
Never work directly on the `main` or `master` branch. Create a descriptive branch name:
```bash
git checkout -b feature/fix-ui-bug
```

### 4. Code and Test
* Write clean, readable code matching the existing style of the Xcode project.
* Test your changes thoroughly on macOS to ensure no existing functionality is broken.

### 5. Submit a Pull Request
Push your branch to your GitHub fork and open a Pull Request against our `main` branch. 

In your PR description, please include:
* A clear explanation of what your code changes accomplish.
* A reference to the Issue number it resolves (e.g., `Closes #12`).
* **The Contribution Agreement Checkbox** (see below).
